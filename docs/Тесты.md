Тестирование в терминале — пошагово
Нужно 2–3 вкладки терминала. Везде порт 8080.

Терминал 1 — сервер
    /tmp/qweb 8080
    Сервер молчит пока работает — это нормально. Все логи отключены по ТЗ.

Терминал 2 — основные тесты:
Шаг 1: PUT — кладём сообщения
# Ожидаем: пустое тело, статус 200
curl -i -XPUT "http://localhost:8080/pet?v=cat"
curl -i -XPUT "http://localhost:8080/pet?v=dog"
curl -i -XPUT "http://localhost:8080/role?v=manager"
curl -i -XPUT "http://localhost:8080/role?v=executive"

Шаг 2: GET — забираем в FIFO-порядке
# Ожидаем: cat, 200
curl -i http://localhost:8080/pet

# Ожидаем: dog, 200
curl -i http://localhost:8080/pet

# Ожидаем: пустое тело, 404 (очередь пуста)
curl -i http://localhost:8080/pet

# Ожидаем: manager, 200
curl -i http://localhost:8080/role

# Ожидаем: executive, 200
curl -i http://localhost:8080/role

# Ожидаем: пустое тело, 404
curl -i http://localhost:8080/role

Шаг 3: Ошибочные запросы
# PUT без параметра v — ожидаем 400
curl -i -XPUT "http://localhost:8080/pet"

# PUT с пустым именем очереди — ожидаем 400
curl -i -XPUT "http://localhost:8080/?v=x"

# GET пустой очереди без timeout — ожидаем мгновенный 404
curl -i "http://localhost:8080/nosuchqueue"

Шаг 4: timeout — GET ждёт сообщение
В терминале 2:
# Зависает на 10 секунд, ждёт сообщение
curl -i "http://localhost:8080/color?timeout=10"

Не прерываем — переходим в терминал 3:
# Отправляем сообщение пока GET ждёт
curl -XPUT "http://localhost:8080/color?v=red"

Терминал 2 должен немедленно вернуть red со статусом 200.

Шаг 5: timeout истекает → 404
# Ждём ровно 2 секунды, никто не шлёт — ожидаем 404
time curl -i "http://localhost:8080/empty?timeout=2"
# real ~2.0s

Шаг 6: FIFO для ждущих получателей
Два GET с таймаутом запускаем почти одновременно (терминал 2 и 3):

Терминал 2:
curl -i "http://localhost:8080/order?timeout=10"
Терминал 3 (сразу после):
curl -i "http://localhost:8080/order?timeout=10"
Потом из любого терминала шлём два PUT:
curl -XPUT "http://localhost:8080/order?v=first"
curl -XPUT "http://localhost:8080/order?v=second"
Ожидаем: терминал 2 получает first, терминал 3 получает second — строго по порядку запросов.

Шаг 7: Graceful shutdown
# Запускаем долгий GET (ждёт 30 секунд)
curl -i "http://localhost:8080/shutdown-test?timeout=30" &

# Через секунду — SIGTERM серверу
sleep 1 && kill -TERM $(pgrep qweb)
Ожидаем: сервер дожидается завершения активного запроса (вернёт 404 по таймауту или сразу если придёт PUT), только потом выходит. Новые соединения не принимает.
